haradaAI

A goal-planning web application inspired by the Harada Method, the goal-setting framework famously used by MLB star Shohei Ohtani when he was just a high school student. The app transforms any ambitious goal into a structured 9x9 grid containing 8 supporting pillars and 64 actionable tasks, giving users a comprehensive roadmap to achieve their objectives. Implemented client-side routing with React Router and configured Vercel rewrites to handle SPA navigation, resolving 404 errors on direct URL access to shared grids.

Frontend --> React | TypeScript | TailwindCSS | React Router
Backend --> Supabase (PostgreSQL database + Edge Functions)
AI Tier --> Groq's Llama 3.3 70B

VoiceCode

A voice-controlled coding assistant that reimagines how we write code by letting your voice do the heavy lifting instead of the keyboard. Originally inspired by a teacher's suggestion (shoutout Mr. Gardner) to build something for physically impaired coders, it's grown into a full-featured voice-driven code editor. Built a flexible regex-based command parser that recognizes natural language variations for navigation commands, allowing users to say "go to," "navigate to," or "jump to" interchangeably. Implemented a dual-mode system that gives users control over how literally the AI interprets their requests (smart vs. strict mode). Planning to evolve this into a full AI-powered development environment that learns your coding style, explains bugs, and provides real-time feedback on your code.

Frontend --> React | TypeScript | TailwindCSS | Vite
Backend --> Node.js | Express
AI Tiers --> Groq's Llama 3.3 70B | OpenAI's GPT-4o-mini

Live demo | Source code


MyNCRep

A civic-information tool built to help North Carolina voters cut through the noise and get the facts that matter. It shows where to vote, what you need, and what each candidate actually stands for. No spin, no party narratives, just information you can use. Created for the Congressional App Challenge by a team of students who realized most new voters don't lack opinions; they lack context. Overcame missing North Carolina polling data in the Voter Information Project (VIP) by implementing fallback logic. Balanced the tradeoff between API-driven data flow and local storage for performance. Designed location features with user privacy and safety as a priority, and established guidelines to ensure all civic data is presented neutrally and verifiably. Planning to expand beyond North Carolina, add real-time legislative updates, build gamified civics learning modules, integrate AI-powered bill summaries, and partner with election boards for verified polling data.

Backend --> Python | Poetry (dependency management)
APIs --> Google Civics API | Vote Smart | NC State Board of Elections (NCSBE)
Mapping + Architecture --> Leaflet.js | OpenStreetMap | Modular design for easy contributor extensions

Live demo | Source code
